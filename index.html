Wearable Sync Integration Manual

Welcome to the Wearable Ingest Interface. You can connect physical watches or custom physiological sensors to compile data inside the BeatSync platform. The backend runs a public ingestion webhook at /api/wearable/ingest.

Transmit Simulated Vital Signs via cURL

Verify network ingress by pushing a JSON payload to the ingest endpoint using curl in your shell:

curl -X POST http://localhost:5000/api/wearable/ingest \
  -H "Content-Type: application/json" \
  -d '{
    "heartRate": 88,
    "systolicBP": 132,
    "diastolicBP": 84,
    "bloodOxygen": 96,
    "glucose": 110,
    "temperature": 36.9
  }'

Executing this will immediately bypass the browser generator loop, triggering the ML predictive scoring models and updating the telemetry dials.

Apple HealthKit Background Ingress Integration

Implement background HealthKit delivery in your iOS Companion App using Swift to send updates from Apple Watch:

import HealthKit

func sendVitalsToCloud(heartRate: Double, spo2: Double) {
    let url = URL(string: "http://localhost:5000/api/wearable/ingest")!
    var request = URLRequest(url: url)
    request.httpMethod = "POST"
    request.setValue("application/json", forHTTPHeaderField: "Content-Type")
    
    let json: [String: Any] = [
        "heartRate": Int(heartRate),
        "bloodOxygen": Int(spo2),
        "systolicBP": 120, // Default baseline if watch lacks sensor
        "diastolicBP": 80
    ]
    
    request.httpBody = try? JSONSerialization.data(withJSONObject: json)
    URLSession.shared.dataTask(with: request) { data, response, error in
        if let err = error { print("Sync failed: \(err)") }
        else { print("Vitals successfully synchronized.") }
    }.resume()
}

Fitbit OAuth 2.0 Real-time Subscription Callback

For cloud-to-cloud sync, authorize with Fitbit API and register a webhook callback pointing to your public server endpoint:

# Python callback endpoint mapping to Fitbit webhook payload
@app.route('/api/fitbit/callback', methods=['POST'])
def fitbit_webhook_callback():
    payload = request.json
    for event in payload:
        # Fetch actual physiological data using event subscription token
        vitals_data = fitbit_client.get_heart_rate_time_series(user_id=event['ownerId'])
        # Map parameters to BeatSync ingest
        requests.post('http://localhost:5000/api/wearable/ingest', json={
            'heartRate': vitals_data['heartRate'],
            'bloodOxygen': vitals_data.get('spo2', 98)
        })
    return '', 204

Universal Wearable Sync Bridge (Terra API)

Authenticate Garmin, Samsung, or Fitbit accounts using Terra's authentication widgets and consume real-time webhooks:

import requests

def handle_terra_webhook(webhook_payload):
    # Map Terra event to physiological vitals
    data_type = webhook_payload.get("type")
    
    if data_type == "vitals":
        data = webhook_payload.get("data", [{}])[0]
        vitals_payload = {
            "heartRate": int(data.get("heart_rate_data", {}).get("summary", {}).get("avg_hr_bpm", 72)),
            "bloodOxygen": int(data.get("oxygen_saturation_data", {}).get("avg_spo2_pct", 98)),
            "systolicBP": 120,
            "diastolicBP": 80
        }
        # Ingest to cloud
        requests.post("http://localhost:5000/api/wearable/ingest", json=vitals_payload)
